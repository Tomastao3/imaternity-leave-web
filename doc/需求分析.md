## 1. 项目背景
- **业务痛点**：不同城市产假与津贴规则差异明显，人工计算复杂；节假日顺延与账户方式（企业/个人）对计算影响大；批量场景高频且易错。
- **目标用户**：企业 HR、普通员工。
- **预期收益**：标准化计算口径、降低人力成本、提升效率与准确性；沉淀规则库与年度节假日计划，支持批量导入导出。

## 2. 项目目标
- **核心**：产假周期与津贴补差的准确计算、结果可视化、可导出。
- **配套**：城市规则维护、员工信息管理、节假日计划、返还规则配置、批量处理。
- **存储**：支持 IndexedDB 与 Postgres 双模式；后端 API 标准化，易于扩展。

## 3. 范围与非范围
- **范围**
  - 产假天数与起止日期计算（含节假日顺延、跨月整月判断）。
  - 津贴、应领、补差、个人社保（整月）计算，导出 PDF/Excel。
  - 城市的产假/津贴/返还/员工/节假日规则管理，支持 Excel 导入导出。
  - 批量导入计算与错误报告。
- **非范围**
  - 复杂鉴权与审批流、对外薪资/报销系统对接。
  - 合同档案与全流程管理。

## 4. 用户与权限
- **HR**：访问“基础数据管理”、“批量处理”、“产假津贴计算”、“节假日”。拥有增删改查。
- **员工**：访问“产假津贴计算”、“智能助手”。仅自助计算与查看。
